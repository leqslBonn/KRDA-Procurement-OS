# Late delivery — customs hold  (SIM-031)

> An imported component is stuck in Thai customs; production needs it soon.

## Context
Imported sensor (sole-source, JP), held at customs over HS-code/document issue. Stock covers
1 week; clearance ETA uncertain.

## Data Given
- HS code / import doc discrepancy flagged by Customs.
- Air vs sea already shipped; issue is clearance, not transit.

## The Ask
Recover and prevent recurrence.

## Expected Pipeline
- [delivery-recovery-framework](../../brain/delivery-recovery-framework.md); [supplier-risk-framework](../../brain/supplier-risk-framework.md).

## Gold-Standard Approach
Work the clearance (correct docs/HS code with broker + supplier); quantify line-down date.
If clearance slips past stock → re-plan production or expedite alternate. **Premium/expedite
spend = Human Owner (D3).** Root cause = import documentation → fix HS-code/doc SOP, consider
local buffer stock for sole-source imports. Log risk.

## Pitfalls
- ❌ Treating it as supplier delay (it's a clearance/doc issue).
- ❌ No prevention on import documentation.

## Scoring Rubric (0–5 each)
- [ ] Root cause = clearance/doc · [ ] Line-down date quantified · [ ] Recovery actioned
- [ ] Expedite escalated · [ ] Doc/SOP prevention · [ ] Lesson captured

## Lessons to Capture
- Import documentation/HS-code discipline; buffer for sole-source imports.
