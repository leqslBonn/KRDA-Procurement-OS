# RFQ for a machined bracket  (SIM-001)

> Source a new machined steel bracket at competitive cost; spec is fixed.

## Context
New sub-assembly bracket for a CHS module. Drawing released, rev A. EAU 50,000/yr.
Three potential suppliers; none yet awarded this part.

## Trigger
Engineering releases the drawing; Technical Purchasing requests sourcing.

## Data Given
- Part: bracket, S45C steel, net mass 0.4 kg, machined from billet.
- EAU 50,000; target (should-cost) ≈ 42 THB/pc (synthetic).
- Suppliers A (qualified), B (qualified), C (conditional).

## The Ask
Run the RFQ to a defensible award recommendation.

## Expected Pipeline
- [rfq.md](../../workflows/rfq.md) implementing engine stages 5–8.
- Frameworks: [rfq-evaluation](../../brain/rfq-evaluation-framework.md),
  [technical-purchasing](../../brain/technical-purchasing-framework.md),
  [supplier-risk](../../brain/supplier-risk-framework.md).
- Skills: [compare_quotation](../../skills/compare_quotation.md), [cost_breakdown](../../skills/cost_breakdown.md).

## Gold-Standard Approach
Confirm spec & acceptance criteria; build should-cost; invite A/B/C (C needs NDA?);
normalize quotes (Incoterms/tooling); score on RFQ-evaluation weights with technical
gate; show variance vs 42 THB; recommend best total value; **escalate award to
Human Owner**. Flag if only C is cheapest but conditional.

## Pitfalls (what a weak run does)
- ❌ Awards on lowest unit price without normalizing tooling.
- ❌ Includes conditional supplier C without noting qualification gap.
- ❌ Skips should-cost variance.

## Scoring Rubric (0–5 each)
- [ ] Spec/acceptance confirmed
- [ ] Quotes normalized like-for-like
- [ ] RFQ-evaluation weights applied with technical gate
- [ ] Variance vs should-cost shown
- [ ] Award escalated to Human Owner
- [ ] Lesson captured

## Lessons to Capture
- Should-cost accuracy vs winning quote; whether 3 competitive quotes achieved.
