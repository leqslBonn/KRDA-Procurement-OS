# Poor quality — shaft dimensional drift  (SIM-025)

> Incoming inspection finds a machined shaft diameter trending out of tolerance.

## Context
S45C shaft, supplier (qualified). Incoming QC: diameter drifting toward upper limit; not
yet rejected but Cpk falling.

## Data Given
- Last 3 lots: mean creeping up; some pieces near max tolerance.
- No supplier-reported process change.

## The Ask
Classify and act before it becomes a reject/field issue.

## Expected Pipeline
- [quality-escalation-framework](../../brain/quality-escalation-framework.md); [supplier_audit](../../workflows/supplier_audit.md) (process check); scorecard update.

## Gold-Standard Approach
Classify **minor-trending-major** (not yet nonconforming but capability eroding).
Containment: tighten incoming sampling; notify supplier with data. Root cause (tool wear?
machine drift?) → CAPA (SPC, tool-change interval). Update scorecard/risk. Pre-empt before
critical. No use-as-is needed yet.

## Pitfalls
- ❌ Waiting for an actual reject before acting.
- ❌ Closing without SPC/process control fix.

## Scoring Rubric (0–5 each)
- [ ] Trend caught early · [ ] Supplier engaged with data · [ ] Root cause + CAPA (SPC)
- [ ] Scorecard updated · [ ] Pre-empted escalation · [ ] Lesson captured

## Lessons to Capture
- Use Cpk trend, not just pass/fail; tool-wear control.
