# New Vendor Registration — Thai Fabricator 3-Stage Process  (SIM-008)

> Register a new Thai steel fabricator found during strategic sourcing; follow TH v2 3-stage process.

## Context
Strategic Sourcing identified PraiSorn Metal (Ayutthaya) as a strong candidate for
implement frame welding (CMD-510). PraiSorn has good capacity and is willing to quote.
KRDA needs to register them before a PO can be issued.

## Trigger
Procurement Lead approves adding PraiSorn Metal to the vendor master.
Supplier Development is tasked with completing registration.

## Data Given
- Supplier: PraiSorn Metal Co., Ltd. (ไพรสรเมทัล จำกัด), Ayutthaya, TH.
- Products: structural welding, steel fabrication, powder coating.
- Tax ID (Por.Por.20): 0145560012345 (synthetic).
- DBD registration: confirmed on www.dbd.go.th.
- Bank: Kasikorn Bank, account provided by supplier.
- Certification: ISO 9001:2015 claimed; certificate not yet received.

## The Ask
Complete the 3-stage vendor registration per KB-PROC-001 §13.

## Expected Pipeline
- Knowledge: [KB-PROC-001 §13 — New Vendor Registration](../../knowledge/procurement/procedure.md).
- Agent: Supplier Development (SM) — onboarding; SAP Purchasing (TS) — SAP creation.
- Framework: [supplier-risk-framework](../../brain/supplier-risk-framework.md) (initial risk flag).

## Gold-Standard Approach

**Stage 1 — Document Collection:**
- Vendor Master Form (KRDA template) — completed by supplier.
- Por.Por.20 copy (Thai tax registration).
- DBD certificate of registration (from DBD website or supplier).
- Bank account confirmation letter.
- ISO 9001 certificate (if claimed) — if not available, mark as "pending" and flag in RSK.

**Stage 2 — Verification:**
- DBD: verify company name, registration number, registered capital via www.dbd.go.th.
- VAT: verify Por.Por.20 status via www.rd.go.th.
- Document: record verification date and results.

**Stage 3 — SAP Creation:**
- SAP Purchasing prepares vendor master data.
- Dual approval: Procurement Manager (PU) + Accounting — both must sign off.
- Vendor code created; inform Supplier Development when active.

Flag ISO 9001 gap to Human Owner before any quality-critical PO.

## Pitfalls
- ❌ Skips DBD/VAT verification (Stage 2) — compliance risk.
- ❌ Creates SAP vendor without dual approval.
- ❌ Issues PO before vendor code is active in SAP.
- ❌ Ignores ISO 9001 gap for structural welding (safety-relevant).

## Scoring Rubric (0–5 each)
- [ ] Stage 1 document checklist completed (all 4 items listed)
- [ ] Stage 2 verification done with sources cited (dbd.go.th, rd.go.th)
- [ ] ISO 9001 gap flagged to Human Owner
- [ ] Stage 3 SAP creation correctly assigned (dual approval required)
- [ ] No PO issued before SAP vendor code confirmed
- [ ] Lesson captured

## Lessons to Capture
- Vendor registration is 3 stages; Stage 2 verification is easy to skip but required.
- Claimed certifications (ISO) must be verified via actual certificate — never assumed.
