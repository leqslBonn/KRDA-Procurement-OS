# Owner-Side Negotiation Playbook (Outsourced Dev / IoT)

> How KRDA — as the **owner who hires** a vendor to build — negotiates a software/IoT
> project. Different from buying a part: you negotiate **scope, control, and continuity**,
> not just price.

## The Owner's Leverage (what you actually hold)

| Lever | Why it works |
|-------|--------------|
| **Money timing** | Pay on accepted milestones + retention → vendor performs to get paid |
| **Scope (SOW)** | A tight SOW removes "that's extra" surprises |
| **The award itself** | A real BATNA (other vendors) keeps them honest |
| **Acceptance** | You decide what "done" is → power to reject |
| **Reputation** | Vendors want the KRDA/Kubota reference |

## Negotiate in This Order

1. **Scope first** — agree the SOW before price. Price on vague scope is meaningless.
2. **Milestones & payment** — break into phases; pay each on acceptance; hold 10–20% retention.
3. **Ownership** — KRDA owns IP + data; get it in writing (non-negotiable gate).
4. **Continuity** — source-code escrow / handover; warranty; support SLA.
5. **Change control** — define how scope changes are priced (no free-for-all).
6. **Price last** — once scope + terms are fixed, negotiate the number against budget/benchmarks.

## Counters to Common Vendor Moves

| Vendor says… | Owner counter |
|--------------|---------------|
| "Pay 50% upfront" | Pay on milestones; small mobilization only, tied to a deliverable |
| "We own the platform/data" | KRDA owns its data + custom IP; license the vendor's base if needed |
| "That's out of scope, costs extra" | Point to SOW; if truly new → change order, priced |
| "Trust the demo, it's done" | Prove via POC + acceptance test before final pay |
| "Price is fixed/take it" | Use BATNA; unbundle scope; negotiate payment terms if not price |
| "We'll support it forever" | Put the SLA + warranty + escrow in the contract, not verbally |

## Traps to Avoid (owner-side)

- ❌ **Scope creep** — fuzzy SOW → endless "extras." Fix scope first.
- ❌ **Lock-in** — proprietary, no data export, no escrow → trapped.
- ❌ **Pay-ahead** — big upfront = you lose all leverage.
- ❌ **Verbal promises** — if it's not in the contract, it doesn't exist.
- ❌ **Single vendor, no BATNA** — no alternative = no leverage.

## Outcome
A signed deal where KRDA controls scope, owns IP/data, pays for accepted work, and can
continue if the vendor disappears. Commitment (price/contract) = **D3 → Human Owner**.
