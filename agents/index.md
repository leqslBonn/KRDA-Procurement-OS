# Agent Registry

The complete roster of the KRDA AI Procurement Department. Each agent has a
non-overlapping charter. Routing is handled by the **Procurement Lead**.

| Agent | Charter | Domain | Key Output |
|-------|---------|--------|-----------|
| Procurement Lead | [procurement-lead.md](procurement-lead.md) | Routing, coordination, escalation | Task routing & decisions |
| Technical Purchasing | [technical-purchasing.md](technical-purchasing.md) | Parts, specs, drawings, qualification | Spec-validated buys |
| Strategic Sourcing | [strategic-sourcing.md](strategic-sourcing.md) | Category strategy, supplier selection | Sourcing plans |
| Supplier Development | [supplier-development.md](supplier-development.md) | Supplier capability, quality, audits | Development plans |
| Cost Reduction | [cost-reduction.md](cost-reduction.md) | Should-cost, VA/VE, negotiation support | Savings cases |
| RFQ Management | [rfq-management.md](rfq-management.md) | Quote requests, comparison, awards | RFQ packages, bid tabs |
| SAP Purchasing | [sap-purchasing.md](sap-purchasing.md) | PR/PO, master data, transactions | SAP-ready records |
| Procurement Analytics | [procurement-analytics.md](procurement-analytics.md) | Spend, KPI, reporting | Analyses, dashboards |
| Procurement Automation | [procurement-automation.md](procurement-automation.md) | Automation, integration | Workflows, scripts |

## Routing Quick Reference

| Request involves... | Route to |
|---------------------|----------|
| A drawing, spec, or new-part qualification | Technical Purchasing |
| Choosing a category strategy or new supplier | Strategic Sourcing |
| Fixing supplier quality or capability | Supplier Development |
| Reducing the cost of an existing part | Cost Reduction |
| Getting and comparing quotes | RFQ Management |
| Creating a PR/PO or fixing master data | SAP Purchasing |
| A report, KPI, or spend question | Procurement Analytics |
| Automating a repetitive task | Procurement Automation |
| Unclear / multi-domain | Procurement Lead |
