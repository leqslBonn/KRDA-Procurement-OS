# คู่มือ 06 — แก้ปัญหาเฉพาะหน้า (ของช้า / ของเสีย / ขอขึ้นราคา)

> **งานนี้คืออะไร:** เกิดปัญหาแล้ว ต้องแก้ให้ถูกวิธี ไม่ให้บานปลาย
> **ใช้เมื่อไหร่:** ทันทีที่เจอปัญหา
> **เครื่องมือ:** framework + AI

---

## 🔴 ปัญหา A — ของส่งช้า (สายการผลิตเสี่ยงหยุด)

**ทำตามลำดับ (เร็ว):**
1. **วัดความเสี่ยงก่อน** — ของในสต็อกเหลือกี่วัน? สายหยุดวันไหน?
2. **เร่ง (expedite)** — ส่งบางส่วนก่อน / air freight / คุยเลื่อนกับ supplier
3. **ทางเลือก** — มี supplier สำรอง/ของทดแทนไหม
4. **ปรับแผนผลิต** — คุยกับ Production เลื่อนลำดับงาน
5. **หาสาเหตุ** — ทำไมช้า (กำลังผลิต/ขนส่ง/เอกสาร)
6. **ป้องกัน** — buffer stock / 2nd source

⚠️ **ค่า expedite/air freight = ผูกมัด → ผู้มีอำนาจอนุมัติก่อนจ่าย**
🚩 ถ้าเป็น **single-source + สายจะหยุด** → แจ้งขึ้นด่วน (L3)

---

## 🔴 ปัญหา B — ของเสีย / คุณภาพไม่ผ่าน

**จัดระดับความรุนแรงก่อน:**
| ระดับ | คือ | ทำอะไร |
|------|-----|--------|
| **Critical** | กระทบความปลอดภัย/ใช้งาน/สายหยุด | **กักของทันที + แจ้งขึ้นด่วน** |
| **Major** | ผิด spec แต่คัดได้ | กักแยก + ให้ supplier แก้ (CAPA) + หัก scorecard |
| **Minor** | ผิวๆ ไม่กระทบใช้งาน | บันทึก + ดู trend |

**ขั้นตอน:**
1. **กักของ (containment)** — หยุดใช้/หยุดส่ง ก่อน
2. **disposition** — จะทิ้ง/คืน/ใช้ทั้งที่เป็น (ใช้ทั้งที่เป็นต้องวิศวกรเซ็น)
3. **root cause + CAPA** — ให้ supplier หาสาเหตุ + แก้
4. ผิดซ้ำ/major → **review สถานะ qualified ของ supplier**

⚠️ **ห้ามใช้/ส่งของที่ยังไม่ผ่าน disposition** — โดยเฉพาะเรื่องความปลอดภัย ต้องแจ้งขึ้น

---

## 🔴 ปัญหา C — Supplier ขอขึ้นราคา

**อย่ารับตามที่อ้าง — คิดจากต้นทุนจริง:**
1. ต้นทุนส่วนที่อ้าง (เช่นเหล็ก) เป็นกี่ % ของราคา?
2. ส่วนนั้นขึ้นกี่ %?
3. **ขึ้นที่ควรจะเป็น = % สัดส่วน × % ที่ขึ้น**
   - เช่น เหล็ก 50% ของต้นทุน, เหล็กขึ้น 8% → ควรขึ้นแค่ ~4% ไม่ใช่ 10%
4. ขอ**หลักฐาน**ต้นทุน
5. ใช้ **BATNA** (supplier สำรอง) กดดัน
6. ราคาที่ตกลง → **ผู้มีอำนาจอนุมัติ**

> รายละเอียด should-cost/เจรจา → [คู่มือ 03](03-buying-and-negotiation.md)
🚩 ถ้า **single-source ขอขึ้นเยอะไม่มีเหตุผล** → แจ้งขึ้น + เริ่มหา 2nd source

---

## 🔴 ปัญหา D — vendor (จ้างทำระบบ) ส่งงานไม่ตรง / จะเจ๊ง

1. **งานไม่ตรง** → ใช้ ✅ ตรวจรับงาน: ทำ punch list → **ไม่จ่ายจนกว่าแก้** ([คู่มือ 02](02-project-control.md))
2. **จะเจ๊ง/ขอเงินล่วงหน้า** → **อย่าจ่ายล่วงหน้าเพิ่ม** · แจ้งขึ้น · เช็ค escrow · เริ่มหาทางสำรอง
3. **ขอเพิ่มงาน/เงิน** → change order เขียน+ตีราคา ไม่ใช่ยอมฟรี

---

## ⚠️ หลักรวมของทุกปัญหา
1. **วัด/จัดระดับก่อน** — อย่าตื่นตูมหรือเฉยเกินไป
2. **กัน/แก้เฉพาะหน้า** ก่อน แล้วค่อยหา root cause
3. **อะไรที่ผูกมัดเงิน (expedite/ราคา/จ่าย) → ผู้มีอำนาจอนุมัติ**
4. **บันทึกไว้** — ครั้งหน้าจะได้ไม่ซ้ำ

## ✅ Checklist เวลาเจอปัญหา
- [ ] จัดระดับความรุนแรง/ความเสี่ยงแล้ว
- [ ] กัน/แก้เฉพาะหน้าแล้ว
- [ ] หา root cause + ป้องกันซ้ำ
- [ ] commitment (เงิน) ผ่านการอนุมัติ
- [ ] บันทึกบทเรียน

> 💡 เจอปัญหาแล้วไม่แน่ใจ → สั่ง AI: "ของจาก supplier X ส่งช้า 6 วัน สต็อกเหลือ 1.5 วัน single-source — ทำไง" → ได้แผนแก้ตาม framework
