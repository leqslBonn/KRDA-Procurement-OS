# คู่มือ 01 — เลือก Vendor (IoT / Smart Farm / จ้างพัฒนา)

> **งานนี้คืออะไร:** มีผู้พัฒนา/บริษัท (vendor) หลายเจ้ามานำเสนองาน (pitch) แล้วคุณต้องเลือกว่าจะใช้เจ้าไหน
> **ใช้เมื่อไหร่:** ก่อนตัดสินใจจ้าง — ตอนกำลังหา/ฟัง vendor นำเสนอ
> **โปรแกรม:** แท็บ 🧮 ประเมิน Vendor + 💵 TCO + 📄 Report

---

## ⏱️ ภาพรวม 3 จังหวะ (ทำตามลำดับนี้)
```
ก่อน pitch → เตรียมเกณฑ์ + คำถาม
   ระหว่าง pitch → ให้คะแนนสด + เช็ค gate + จด red flag
      หลัง pitch → เทียบ + TCO + สรุป report เสนอหัวหน้า
```

---

## 🟦 STEP 1 — ก่อนวันฟัง pitch (เตรียมตัว)

**1.1 เขียนสั้นๆ ว่าต้องการอะไร** (1 ย่อหน้า)
- ระบบนี้แก้ปัญหาอะไรของ KRDA? เช่น "เซนเซอร์ความชื้นดิน + dashboard สำหรับไร่ทดลอง"
- ขนาดงาน? เช่น กี่จุด กี่แปลง กี่คนใช้
> ถ้างานใหญ่/ผูกพันยาว ควรเขียนเป็น SOW เต็ม → ดู [คู่มือ 02](02-project-control.md)

**1.2 เปิดโปรแกรม → แท็บ 🧮 ประเมิน Vendor**
- ดูเกณฑ์ 11 ข้อที่มีอยู่ (problem-fit, tech, team, traction, scalability, integration, **data ownership⛔**, TCO, **viability⛔**, lock-in, support)
- ⛔ = "gate" (เงื่อนไขบังคับ) — 2 ข้อนี้ห้ามพลาด: **ใครเป็นเจ้าของข้อมูล** และ **บริษัทจะเจ๊งไหม**

**1.3 ปรับ weight (น้ำหนัก) ให้ตรงงาน** (ถ้าจำเป็น)
- ค่าเริ่มต้นใช้ได้เลยสำหรับงาน smart farm ทั่วไป
- ถ้างานเน้นอย่างใดเป็นพิเศษ ปรับได้ **แต่รวมต้อง = 100%** (แถบบนจะเตือน)
- 📌 **ตั้ง weight ก่อนฟัง pitch เสมอ** — กันเผลอเข้าข้างเจ้าที่ชอบ

**1.4 เตรียมคำถามที่ต้องถามทุกเจ้า** (จดใส่กระดาษ/มือถือ)
คำถาม 5 ข้อบังคับ:
1. "ข้อมูลที่ระบบเก็บ (ข้อมูลฟาร์ม) **ใครเป็นเจ้าของ**? เราขอ export ได้ไหม?"
2. "ถ้าบริษัทคุณ**เลิกกิจการ** ระบบเราจะเป็นยังไง? มี source-code escrow ไหม?"
3. "ขอดู**ลูกค้าจริงที่ใช้อยู่** / demo ระบบจริง (ไม่ใช่สไลด์)?"
4. "**เชื่อมกับระบบเดิม**ของเราได้ไหม? มี API ไหม?"
5. "ค่าใช้จ่าย**รวม 3-5 ปี** เท่าไหร่? (ไม่ใช่แค่ราคาแรกเข้า)"

---

## 🟩 STEP 2 — ระหว่างฟัง pitch (ให้คะแนนสด)

**2.1 เปิดแท็บ 🧮 → แก้ชื่อเจ้า A/B/C เป็นชื่อจริง**
- ช่องบนสุด แก้ "A · AgriSense" เป็นชื่อ vendor จริง

**2.2 ให้คะแนนแต่ละเกณฑ์ 1–5** (ระหว่าง/หลังฟังแต่ละเจ้า)
- 1 = แย่/ไม่มี · 3 = พอใช้ · 5 = ดีมาก
- ให้ตามที่**เห็นจริง** ไม่ใช่ตามความประทับใจ pitch
- ตัวอย่าง: เจ้าไหนมีแต่สไลด์ ไม่มี demo → tech maturity ให้ 2

**2.3 จับ red flag (ธงแดง) — จดไว้**
- 🚩 มีแต่สไลด์ ไม่มี demo จริง / ไม่มีลูกค้าจริง
- 🚩 เลี่ยงตอบเรื่องเจ้าของข้อมูล
- 🚩 บริษัทเล็กมาก คนเดียว/เงินทุนน้อย
- 🚩 ผูกขาด ย้ายออกไม่ได้
- 🚩 พูดแต่ราคาถูก ไม่ตอบเรื่องดูแลหลังขาย

**2.4 ดูผลสด**
- คอลัมน์ล่างจะโชว์ **คะแนนรวม + Gate (✓/✗) + สรุป (แนะนำ★/สำรอง/ตัด)**
- ถ้าเจ้าไหน **gate ✗** = ถูกตัดทันที ไม่ว่าคะแนน/ราคาดีแค่ไหน

---

## 🟨 STEP 3 — หลัง pitch (เทียบต้นทุน + สรุป)

**3.1 ไปแท็บ 💵 TCO — ใส่ต้นทุนแต่ละเจ้า**
- ใส่: hardware, ค่าติดตั้ง, **ค่าบริการรายปี**, support รายปี, ค่าย้ายออก
- ตั้งจำนวนปี (เช่น 5)
- ระบบบอก **TCO รวม 5 ปี** ของแต่ละเจ้า + ใครถูกสุด
- 📌 **จุดสำคัญ:** เจ้าที่ราคาแรกถูก อาจรวม 5 ปีแพงกว่า (ค่ารายปี + ค่าย้ายออก)

**3.2 กลับแท็บ 🧮 → กด "สร้าง Report"**
- ระบบเด้งไปแท็บ 📄 Report — ได้บันทึกข้อความสรุป (ภาษาไทย/EN)
- มี: ข้อเสนอแนะ (เลือกเจ้าไหน) + ตารางคะแนน + สิ่งที่ขออนุมัติ

**3.3 กด 🖨️ พิมพ์ / PDF → ส่งหัวหน้า**
- ได้ไฟล์ PDF พร้อมส่ง/ปริ้น เสนอผู้มีอำนาจอนุมัติ

**3.4 เซฟเคสไว้** (กด "เซฟเคสนี้") — ดูย้อนหลังได้ที่แท็บ 🗂️

---

## ⚠️ กับดักที่พบบ่อย (อย่าทำ)

| ❌ อย่า | ✅ ให้ทำ |
|--------|---------|
| เลือกเจ้าถูกสุดทันที | เช็ค gate ก่อน — ถูกแต่ยึด data/จะเจ๊ง = ตัด |
| เชื่อ demo สวยว่า "ทำเสร็จแล้ว" | ขอ POC ทดลองจริงก่อนซื้อ (ดู [02](02-project-control.md)) |
| ปรับ weight หลังเห็นราคา | ตั้ง weight ก่อนฟัง pitch |
| ตัดสินคนเดียวในหัว | ให้คะแนนทุกเกณฑ์เป็นระบบ กันลืมมุมสำคัญ |
| ลืมถามเรื่องเจ้าของข้อมูล | ถาม + ต้องได้คำตอบว่า KRDA เป็นเจ้าของ |

---

## ✅ Checklist ก่อนเสนอหัวหน้า
- [ ] ให้คะแนนครบทุกเจ้า ทุกเกณฑ์
- [ ] เจ้าที่แนะนำ **ผ่าน gate ทุกข้อ** (data + viability)
- [ ] ทำ TCO 5 ปีเทียบแล้ว
- [ ] มี red flag จดไว้ (ถ้ามี)
- [ ] สร้าง report + PDF แล้ว
- [ ] ในข้อเสนอมี "ทำ POC ก่อน" (ถ้าเจ้าที่เลือกยังไม่เคยพิสูจน์ที่ขนาดเรา)
