## 📖 หลักการที่ใช้ทุกเล่ม (จำ 5 ข้อนี้)

1. **จ่ายตามงานที่ตรวจรับ ไม่จ่ายล่วงหน้าก้อนใหญ่** — เงิน = อำนาจคุมเดียวที่คุณมี
2. **KRDA ต้องเป็นเจ้าของข้อมูล + IP เสมอ** — ห้ามให้ vendor ยึด
3. **ราคาถูกไม่พอ** — ต้องผ่าน "เงื่อนไขบังคับ (gate)" ก่อน
4. **เขียนให้ชัดก่อนเริ่ม** (SOW/เกณฑ์) — กันเถียงกันทีหลัง
5. **คุณไม่เซ็นอนุมัติเอง** ถ้าเกินอำนาจ — ส่งขึ้นผู้มีอำนาจ (ดู [05](05-documents-approval.md))

## รูปแบบทุกเล่ม
แต่ละเล่มมี: **งานนี้คืออะไร → ใช้เมื่อไหร่ → ต้องเตรียมอะไร → ขั้นตอนละเอียด (1-2-3) → กับดักที่พบบ่อย → checklist ก่อนจบ**

## หมายเหตุสำคัญ (ยังรอยืนยัน)
- ⚠️ **เกณฑ์วงเงินอนุมัติ (ต่อ PO vs ต่อปี)** ยังรอ KRDA ยืนยัน — ดูเล่ม [05](05-documents-approval.md)
- ⚠️ **rate ต้นทุนใน Should-cost** เป็นค่าประมาณ ต้องแทนด้วยราคาจริง — ดูเล่ม [03](03-buying-and-negotiation.md)
