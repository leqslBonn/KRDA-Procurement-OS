## Must-Have Terms

| # | Term | In contract? | Notes |
|---|------|:-----------:|-------|
| 1 | **Scope = SOW** (attached, controls "done") | ☐ | [sow-requirement](sow-requirement.md) |
| 2 | **IP ownership** — KRDA owns custom deliverables & source | ☐ | not "license back" |
| 3 | **Data ownership** — KRDA owns operational/farm data; export anytime; deletion on exit | ☐ | the #1 trap |
| 4 | **Payment on milestone acceptance** (not upfront) | ☐ | tie to [acceptance-checklist](acceptance-checklist.md) |
| 5 | **Retention** — 10–20% held to warranty end | ☐ | |
| 6 | **Change-order process** — scope change = written + priced | ☐ | cost-creep shield |
| 7 | **Warranty** — free defect fix for N months | ☐ | |
| 8 | **Support / SLA** — response time, uptime, penalties | ☐ | |
| 9 | **Late / non-performance penalty** (liquidated damages) | ☐ | |
| 10 | **Source-code escrow / handover** if vendor ceases | ☐ | startup continuity |
| 11 | **Confidentiality / NDA** | ☐ | [nda workflow](../workflows/nda.md) |
| 12 | **Exit / transition** — handover, no lock-in, data out | ☐ | |
| 13 | **Acceptance defines payment** — clear criteria | ☐ | |
| 14 | **Sub-contracting** disclosed & controlled | ☐ | |
| 15 | **Liability / indemnity** reasonable | ☐ | legal review |

## Red-Flag Terms (push back)
- 🚩 Vendor owns the data or the IP.
- 🚩 Large upfront payment, no milestones.
- 🚩 Auto-renewal / price escalation uncapped.
- 🚩 No exit / data-export clause (lock-in).
- 🚩 No escrow for a startup-built system.
